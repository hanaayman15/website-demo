🔒 POST /api/auth/refresh

✅ ENDPOINT FULLY IMPLEMENTED AND VERIFIED
POST http://127.0.0.1:8001/api/auth/refresh
Purpose: Refresh expired access token using refresh token.

Token Rotation: Returns new access token AND new refresh token. Old refresh token is invalidated (rotation security pattern).

📥 Request Body

{ "refresh_token": "string" }

📤 Response (200 OK)

Schema: TokenPair

✓ access_token: string [REQUIRED]
✓ refresh_token: string [REQUIRED] ← New token generated
✓ token_type: string [REQUIRED]
✓ access_token_expires: integer [REQUIRED] ← Seconds until expiration
✓ user_id: integer [REQUIRED]
✓ role: string [REQUIRED]

🔐 Security Features

  • Token Rotation: Each refresh generates a new refresh token
  • Old Token Revocation: Previous refresh token becomes invalid
  • Database Tracking: RefreshToken table stores all issued tokens
  • Revocation Check: Validates tokens haven't been revoked before use
  • OWASP Compliant: Follows security best practices
